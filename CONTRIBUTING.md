# Hướng dẫn đóng góp / Contributing to Pilo Masterkit 🚀

Trước hết, cảm ơn bạn đã cân nhắc đóng góp cho `Pilo Masterkit`! (First off, thank you for considering contributing to `Pilo Masterkit`!)

## 🌈 Quy tắc ứng xử / Code of Conduct

Dự án này và tất cả những người tham gia đều được điều chỉnh bởi [Quy tắc ứng xử](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## 🛠️ Làm thế nào để đóng góp? / How Can I Contribute?

### Báo cáo lỗi / Reporting Bugs

- Kiểm tra [Issues](https://github.com/Arter2507/pilo-masterkit/issues) để xem lỗi đã được báo cáo chưa. (Check the Issues to see if the bug has already been reported.)
- Nếu chưa, hãy tạo issue mới bằng mẫu **Bug report**. (If not, create a new issue using the Bug report template.)

### Đề xuất cải tiến / Suggesting Enhancements

- Kiểm tra [Issues](https://github.com/Arter2507/pilo-masterkit/issues) để xem đã có ai đề xuất chưa. (Check the Issues to see if someone else has already suggested it.)
- Nếu chưa, hãy tạo issue mới bằng mẫu **Feature request**. (If not, create a new issue using the Feature request template.)

### Pull Requests

1. **Fork** repository về tài khoản của bạn.
2. **Clone** bản fork về máy cục bộ.
3. **Tạo branch mới** cho công việc của bạn (ví dụ: `feat/new-rule-kit`).
4. **Thực hiện thay đổi** và thêm test nếu có thể.
5. **Đảm bảo các test đều vượt qua** (`npm test`).
6. **Commit thay đổi** sử dụng [Conventional Commits](https://www.conventionalcommits.org/).
7. **Push** branch lên bản fork của bạn.
8. **Gửi Pull Request** vào branch `main` của repository gốc.

## 🤝 Tiêu chuẩn / Standards

- **Conventional Commits**: Mỗi commit phải tuân thủ `feat:`, `fix:`, `docs:`, v.v.
- **Hỗ trợ song ngữ (Bilingual Support)**: Tất cả tài liệu phải được cung cấp bằng cả **Tiếng Việt** và **Tiếng Anh**.
- **Bảo mật (Security)**: Tuyệt đối không commit bí mật hoặc dữ liệu nhạy cảm. Mọi đóng góp đều được quét bởi hệ thống bảo mật nội bộ.

## 📝 Quy trình đánh giá PR / PR Review Process

- Mọi PR đều cần ít nhất một sự phê duyệt từ người duy trì.
- Các kiểm tra CI phải vượt qua.
- Chúng tôi có thể yêu cầu thay đổi hoặc làm rõ trước khi merge.

Cảm ơn sự hỗ trợ của bạn! / Thank you for your help!
